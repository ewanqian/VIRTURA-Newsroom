# 从 MR 热到 Web-first

过去一段时间，我们也曾尝试把 Apple Vision Pro、混合现实展示、空间计算界面当作一个重要入口来思考。但越往下推，越能确认一件事：**设备不是需求，媒介不是需求，炫技也不是需求。**

真正的需求往往更朴素：

- 我怎样更快理解一个复杂系统？
- 我怎样更快练会一个工作流？
- 我怎样把知识从“看过”变成“会用”？
- 我怎样让团队在更低成本下推演一个项目？
- 我怎样把已有资料组织成一个能被别人真正进入的系统？

如果这些问题没有被解决，那么把内容搬进头显，也往往只是在更贵的设备里重复原来的混乱。

## 我们现在更关心的，不是沉浸，而是部署

VIRTURA 过去关于“感知迁移”的讨论，容易被误读成“把内容迁移到新的设备里”。但今天我们更愿意把它理解为：**把复杂知识、项目逻辑、空间方法和工作流经验，迁移为一种更容易进入、更容易练习、更容易协作的组织方式。**

这意味着，真正值得优先推进的，不是大而全的 MR 展示，而是三类更基础、也更有复利价值的系统：

### 1. 交互出版物
不是 PDF，不是 PPT，也不是视频教程，而是一种可点击、可展开、可跳转、可重组的阅读系统。知识在这里不再只是被陈列，而是被组织成可以进入的路径。

### 2. 训练模拟器
不是把知识讲一遍，而是让学习者在限制条件、节点选择、任务目标与即时反馈中“部署知识”。这类系统更接近我们所说的“智力装备”。

### 3. 项目推演器
把展览、演出、教学、文旅或数字内容项目中的判断逻辑拆解出来，形成可比较、可复盘、可协作的推演系统。它不一定长得像一个游戏，但它需要具备游戏一样的可练习性。

## Web-first，不是妥协，而是更接近问题本身

网页端的优势，不只是分发便宜。更重要的是，它天然适合：

- 低摩擦进入
- 版本更新
- GitHub 协作
- Agent 调用
- 内容节点化组织
- 训练任务系统
- 研究文档与产品原型并行生长

当这些层成立之后，MR 反而更容易找到自己的正确位置：它不再承担“证明这个方向存在”的任务，而成为一种已有系统成立后的展示加成、体验加成和空间 bonus。

## 这条线接下来会往哪里走

我们当前更愿意把重点放在以下几类原型：

- 研究查看器
- 节点化知识阅读器
- 训练模拟器
- 项目推演器
- 数字舞台与 Digital Library 的网页工具层

如果它们能跑通，后面的沉浸式投影、展览装置、空间观看层、MR 展示层，都会更自然地长出来。因为那时内容与动线已经成立，设备只是在顺水推舟。

## 一句话结论

**MR 不是主需求，感知迁移也不是头显。**  
**真正值得长期建设的，是 Web-first 的可进入内容系统、可游玩智力装备、训练模拟器与项目推演器。**
