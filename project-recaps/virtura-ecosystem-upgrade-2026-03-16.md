# VIRTURA Ecosystem Upgrade 2026-03-16

- date: 2026-03-16
- project: VIRTURA ecosystem upgrade
- version: first public restructuring wave
- related repo: VIRTURA-Collective / VIRTURA-SpacePort / VIRTURA-Newsroom / portfolio
- author: Ewan Qian

## What Changed

这次升级的目标，不是新建更多仓库，而是把已经存在的仓库角色明确下来。

当前公开结构已经固定为:

- `portfolio`: 个人入口
- `VIRTURA-Collective`: 团队主入口
- `VIRTURA-SpacePort`: 公共导览前台、知识网络主宿主、旧档案源宿主
- `VIRTURA-Newsroom`: 发布出口
- `SceneForge`: 工具与 Digital Stage 实验主线

## Why This Upgrade Was Needed

升级前最大的问题不是内容少，而是角色混用。

- 团队本体和公共前台没有分开
- 知识网络、档案、作品、研究和工具线相互挤占
- 外部读者很难判断从哪里进入

## What Was Built In This Wave

- `VIRTURA-Collective` 作为团队主入口正式建立
- `Drop Flow`、`TIMER`、`Observation and Symbiosis` 进入团队仓的作品结构
- `Activities` 作为沙龙、工作坊和时间线入口被建立
- `knowledge-network` 顶部导览完成重写
- 三大入口首页的阅读流线完成一轮优化

## What Is Still In Progress

- `portfolio` 首页仍需继续压缩和下沉长内容
- `organization/works` 仍然是旧档案强宿主，需要继续减轻初访负担
- `VIRTURA-Newsroom` 还需要补更多正式文章和项目复盘

## Current Boundary

这次升级最重要的一条边界是:

`VIRTURA-Collective` 已经接住团队本体，但 `VIRTURA-SpacePort` 不能被误理解成空壳。

它仍然必须保留知识网络主宿主和旧档案源宿主角色。
