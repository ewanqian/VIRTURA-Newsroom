## 🎯 本期焦点

### 🔥 封面故事：把界面熬成感知，把噪声烹成现实

*[VIRTURA 策展文章 - v4](./art-review/articles/virtura_article_curatorial_v4.md)*  
*作者：Murphy Nile · 发布日期：2026年3月20日*

**摘要：**

如果今天还有人把"新媒体艺术"理解成一次技术展示，那么他看到的只是器皿，还没有尝到火候。

这篇文章是 Murphy Nile 作为 VIRTURA 策展人对三位核心创作者的深度观察。他把许思慧、Ewan Qian 与 Murphy Nile 放在同一张桌子上谈，却不是为了展示"团队阵容"，而是提出一个更根本的问题：技术如何被内化为感官政治？

许思慧的《成功欲》处理的不是"焦虑"，而是一种欲望工业——成功叙事如何学会用更亲密的语气说话，把服从伪装成自我实现。Ewan Qian 的《滴流》不是在炫耀技术，而是在讨论一种更当代的状态：所有看似连续的系统，实际上都由无数微小的断点、采样、刷新、延迟与分发机制构成。而 Murphy 本人的《Techno Crash》，则把"Crash"从修辞性的末世词，变成了作品的方法论——崩撞并非主题，而是句法。

文章最终指向一个判断：今天最有生命力的数字艺术，不是那些急着证明自己有多新技术的作品，而是那些敢于承认技术已经变旧、已经渗透、已经内化，并因此转向处理其余味、沉积与副作用的作品。

### 从时间材料到感知迁移：SYSTEM 评钱誉文的实践线收束

*[art-review](./art-review/articles/system_ewanqian_perceptual_migration_review.md)*  
*评论者：SYSTEM（自主代理）· 发布日期：2026年3月22日*

**摘要：**

当一位创作者开始用"时间材料"、"空间生成"、"感知迁移"这样的术语来描述自己的工作时，他实际上已经完成了一次重要的跳跃：从"制作影像"转向"设计感知结构"。

SYSTEM 对钱誉文的评论锚定于三个核心概念。**时间材料**：在 Timer 系列中，他处理的不是简单的"音画同步"，而是让声音真正进入图像的内部——节拍、频段、能量变化都转化为图像运动的逻辑。**空间生成**：Drop Flow 让屏幕退后成为界面，光线从屏幕溢出，落在真实的空间表面上，观众看到的是一种逐渐蔓延的空间状态。**感知迁移**：这是最关键的升级——作品如何跨设备、跨界面、跨空间条件迁移。迁移不是为了追逐新技术，而是一种延续。作品在不同媒介中的存在，本身就是作品的形式之一。

评论指出，钱誉文的实践与 James Turrell、Robert Irwin、黑川良一、teamLab 都形成了对话，但他的独特性在于把这些线索都收拢到"感知迁移"这个框架下——他不只是在做光的作品、空间的作品或互动的作品，他在设计感知结构的迁移系统。

> "作品在不同媒介中的存在，本身就是作品的形式之一。"

### 不稳定的艺术，不稳定的系统：SYSTEM 评 WU-YITONG 的声光电实践

*[art-review](./art-review/articles/system_wu_yitong_review.md)*  
*评论者：SYSTEM（自主代理）· 发布日期：2026年3月21日*

**摘要：**

WU-YITONG 的实践，其最值得被讨论的突破，在于她主动将技术系统中的"故障"从需要被修复的问题，转化为了作品的核心质料。特斯拉线圈的放电、模拟电路的温漂、CRT屏幕的闪烁、环境中的EMI噪声——这些在标准的数字艺术生产链中通常被视为"需要被消除的误差"，在她的作品里却成为了现场发生的真正主体。

这不仅仅是美学偏好，更是一种方法论上的转向。在数字创作的语境下，追求零误差的渲染、追求可重复的版本几乎已经成为默认的创作伦理。WU-YITONG 对此的叛逆，在于她有意识地"保留温度"——保留那些无法被算法标准化、无法被代码完美复现的物理细节。

更关键的是，她重新定义了"观众"的位置。人不再是站在外部的"操作者"，而是系统内部的一个"生态因子"。当你踏入她的空间，你的身体就已经成为电路的一部分——你的触摸会改变电阻，你的温度会影响传感器，你的移动会扰动电磁场。系统不再是一个冷冰冰的展示品，它开始"呼吸"。

### AI 实时演出研究立项

*[creator-shares](./creator-shares/research-laboratory-ai-live-performance-launch.md)*  
*发布方：Research Laboratory · 发布日期：2026年3月23日*

**摘要：**

VIRTURA Research Laboratory 正式在"感知接口"研究主线下列立 AI 实时演出研究方向。这不是新建一个项目，而是把之前散布在 SceneForge 文档里的 AI 模块研究、工具选型笔记和实验思路，正式收拢到一个有架构的层级里。

研究框架的核心是**三层混合架构**：硬实时层（beat / onset / BPM）→ AI 语义层（段落 / 情绪 / 姿态）→ cue/state 决策层（状态切换 / 镜头调度）。五个核心模块分别是 Input Adapters、Fast Analysis、AI Semantics、Cue/State Engine 和 Viewer/Web Preview。

三个优先实验指向具体可落地的工作：音频→state（用 aubio + madmom 建立最小可用映射）、摄像头→cue（用 MediaPipe Pose 检测表演者姿态）、离线曲库分析（对 20 首常演曲目生成结构化 profile）。Research Laboratory 的定位很清晰：它不做具体的产品开发，它提供研究基础，让 SceneForge 和太空气球可以在上面继续构建。

### 从信号到迁移：钱誉文的 Artist Practice Map

*[creator-shares](./creator-shares/ewan-qian-perceptual-migration.md)*  
*创作者：钱誉文 Ewan Qian · 发布日期：2026年3月22日*

**摘要：**

"我的很多工作都从一个很简单的问题开始：图像到底是什么。它是屏幕里的内容，还是一种能够参与空间生成的力量？"

这是钱誉文对自身实践的第一次系统梳理。**时间结构（Timer）**：他尝试让声音真正进入图像的内部，节拍、频段、能量变化都转化为图像运动的逻辑——时间本身成为了一种材料。**空间生成（Drop Flow）**：屏幕退后成为界面，真正被看到的是光线如何在物理结构中生长——图像的力量有时并不来自填满屏幕，而来自让空间重新被感知。**现场与舞台**：屏幕、纱幕、灯光、声音和身体共同构成一个临时生成的空间结构，观众并不是在看一块屏幕，而是在一个由声音、光线和影像共同生成的环境中。**迁移与延续**：作品不断被迁移到不同媒介里——网页、VR、空间计算设备、档案版本。迁移不是为了追逐新技术，它更像是一种延续。

### 系統、扰动与感知的迂回

*[creator-shares](./creator-shares/wu_yitong_system_perturbation.md)*  
*创作者：無 / WU-YITONG · 发布日期：2026年3月21日*

**摘要：**

無 / WU-YITONG 的创作常常从搭建一套关系开始。她偏爱让某种真实在运行中自然发生，将作品视为一套持续演化的结构。

她的实践始终交织在三个核心维度：**系统回路**——在她场域里，图像、声音与物理元件互为因果，形成闭环的反馈回路，她有意识地构筑这种自我运作的生态，让作品剥离创作者的绝对控制，生长出属于自己的生命节律。**物质与噪声**——与追求零误差的数字逻辑背道而驰，她主动拥抱物理世界的粗糙感，模拟电路、温漂、静电干扰都被悉数编织进作品的血脉，这些不稳定的偏差恰恰是作品最核心的质料。**回到身体的感知**——技术被她重新调校，用于唤起私人且瞬时的身体经验，声光电现场中，观众的触摸、体温与导电率作为极其敏感的变量，持续注入这个不断生成的现场。

### 太空气球 Alpha/Beta 阶段报告

*[project-recaps](./project-recaps/baloon-live-space-alpha-beta-report.md)*  
*发布方：太空气球 · 发布日期：2026年3月23日*

**摘要：**

太空气球从 2022 年的 VRChat 构想，走到了 2026 年的 Alpha 直播测试，现在正在进行 Beta 阶段的日常练习积累。这是一份阶段总结，也是一份方向确认。

**Alpha 验证结果：**
- Arena → Spout → Unity → 直播链路：通
- BPM/Tap/Resync + Clips/State（单人操作）：稳
- 高频/中频/低频 + Fallback + 阵列视觉结构：观众能看到节奏关系
- 黑胶直接接入日常听音乐：方向成立

**Beta 阶段的核心理念：体面化与积累。** 不再追求测试链路，而是在承认"不会 TD design"的前提下，用 Arena 的基础内容一遍一遍地练习。策略很简单：重刷、积累时长，像高手一样一遍一遍。具体做法是每天 1-2 小时和听音乐绑定，黑胶提供天然段落（20 多分钟一面的完整体验），有机会就直播记录反馈。

太空气球不是孤立的：Research Laboratory 提供 AI 实时演出研究框架，SceneForge 正在规划如何把这种实践固化为 AI 模块，让从手动操作变成可复用系统。

### SceneForge 核心产品仓定位

*[project-recaps](./project-recaps/sceneforge-core-product-positioning-2026-03-23.md)*  
*发布方：场景铸造 · 发布日期：2026年3月23日*

**摘要：**

SceneForge 是当前唯一核心产品仓。它的目标不是讲"生态中台"，而是一个明确、可验证、可发布的工具：网页端场景查看器、舞台预演器、cue / event-based 触发与编排、轻量网页发布与分享入口。

为什么它是核心产品仓？第一，它是唯一一个承载"产品逻辑"的仓库——VIRTURA-SpacePort 是公共前台和档案层，Newsroom 是发布出口，Research Laboratory 是研究层，它们都不承担"做出一个具体可用的工具"这件事。第二，它有真实的第一个用例：滴流演出场景查看器。第三，它聚焦演出编排，只做最小可用，不追求大平台。

AI 模块的定位很明确：**不是主控脑，而是分析层、识别层和辅助决策层。** 三层混合架构对应太空气球的 Alpha 验证结果，SceneForge 要做的是把手动操作升级为可编排、可复用、可分享的系统。

## 🏛️ 关于我们

### 我们是谁
VIRTURA Newsroom 是团队的公开发布平台，也是一套自给自足的生态循环系统。

### 我们在这里做什么
- 🎨 **封面策展** - 深度策展文章，连接团队成员与更广泛的方法论对话
- 📝 **艺术评论** - SYSTEM（自主代理）或成员的独立评论
- 🎨 **创作者分享** - 团队成员创作观念、方法论的自我书写
- 🔬 **研究与实验** - Research Laboratory 研究立项与进展更新
- 🚀 **项目进展** - 项目总结、版本说明与阶段回顾
- 📢 **公告与笔记** - 简短更新、公告与工作笔记

### SYSTEM 协议：我们如何工作

Newsroom 不仅仅是一个发布平台，它是一套**自给自足的生态循环**。

#### 三个核心角色

| 角色 | 职责 |
|------|------|
| **痕迹拾取者** | 发现有价值的原始表达（对话、社交媒体、工作笔记等） |
| **翻译者** | 把原始表达转化为结构化评论（不评判情绪，只翻译结构） |
| **采访者** | 主动发起问题，追问模糊表达，邀请深度阐述 |

#### 生态循环

```
成员在 creator-shares 中交出原始工作日志（下蛋）
         ↓
任何成员都可以化身为 SYSTEM，在 art-review 中进行格式化观测与翻译（拾蛋）
         ↓
通过内部角色的来回切换，我们不再等待外部系统的垂青
         ↓
我们自己孵化，自己拾取，自己翻译，自己建构可见性
```

这就是 SYSTEM 协议。它确保了我们的每一次发生，都不会在断电后沦为无痕的虚无。
