# 路线更新：Web-first / MR-optional

## 核心判断

**设备不是需求本身。**

我们不再把 MR 设备或头显界面视为优先需求，而是把重点转向：

- 更快理解
- 更快训练
- 更快协作
- 更快推演
- 更快复盘

## 新优先级

### P0：网页端交互出版物 / 节点化阅读器
把知识网络、项目逻辑和案例结构转成可点击、可进入、可验证的阅读系统。

### P1：训练模拟器 / 智力装备
在节点化阅读器之上，进入"可游玩、可挑战、可评分"的阶段。

### P2：项目推演器
把真实项目经验结构化，转成可以协作、比较、复盘的系统。

### P3：MR / Vision Pro 展示层（可选）
不取消，但不再作为主线。只有当前三层成立之后，MR 才值得接入。

## 为什么网页端优先

- **更低摩擦**：网页比头显更容易打开、传播、协作、版本化
- **更接近真实需求**：人们首先需要"把事情搞清楚"，然后才是"把事情沉浸化"
- **更易被 Agent 调用**：适合 GitHub / Agent / 文档系统引用
- **更能形成复利**：一个网页节点系统可以继续长成交互出版物、练习器、模拟器、项目推演器、MR 展示层
